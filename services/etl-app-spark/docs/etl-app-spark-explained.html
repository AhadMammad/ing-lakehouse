etl-app-spark

An object-oriented PySpark medallion pipeline that ingests the auth identity-service database into an Apache Iceberg lakehouse (Nessie REST catalog + RustFS/S3), across three isolated layers — auth_bronze auth_silver auth_gold — with a Type-2 slowly-changing dim_user.

1. Overview & purpose

The wider ing-lakehouse already lands two sources (payments, rideon) via a PyIceberg/Polars app and dbt-on-Trino. etl-app-spark is a third, fully Spark-native pipeline for a new auth source. Its job is to demonstrate industry-standard medallion practice on a real Spark standalone cluster:

  • Bronze — land raw OLTP tables as-is + provenance meta columns.
  • Silver — cleaned, de-duplicated, conformed tables (SCD1 upsert).
  • Gold — a Kimball star schema with a Type-2 dim_user dimension.

Everything is OOP: one small class hierarchy (SparkJob → Bronze/Silver/Gold) plus thin config, session, catalog, and meta helper modules. A single entry point maps --job/--table/--date to the right class.

Design choices baked in: isolated auth_* namespaces (no collision with the shared bronze/silver/gold), the Iceberg REST catalog against Nessie (no nessie-spark-extensions needed), and client deploy-mode so the driver runs inside spark-master while executors run on the workers.

4. How a job is submitted

The Airflow image has no JDK/Spark client, so each task is a BashOperator that docker execs spark-submit into the running spark-master container (through the docker-socket-proxy). The code and all jars are baked into the custom image, so master and both workers are identical — required for client deploy-mode.

services/airflow/dags/auth_etl.py — the submit command
def spark_submit(job: str, table: str) -> str:
    return (
        f"docker exec {_ENV_FLAGS} {SPARK_CONTAINER} "
        f"/opt/spark/bin/spark-submit "
        f"--master spark://spark-master:7077 --deploy-mode client "
        f"--name auth_{job}_{table}_{{{{ ds }}}} "
        f"{APP} --job {job} --table {table} --date {{{{ ds }}}} --run-id '{{{{ run_id }}}}'"
    )

{{ ds }} (logical date) and {{ run_id }} are Airflow templating. The DAG runs bronze → silver → gold, with max_active_tasks=1 so the small cluster runs one job at a time.

6. session.py — the SparkSession factory

Builds a session configured for the Iceberg REST catalog. Credentials are set programmatically here (kept out of the baked config file); everything else is in spark-defaults.conf.

class SparkSessionFactory:
    @staticmethod
    def build(app_name: str) -> SparkSession:
        cat = config.CATALOG_NAME            # "nessie"
        builder = (
            SparkSession.builder.appName(app_name)
            .config("spark.sql.extensions",
                    "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions")
            .config(f"spark.sql.catalog.{cat}", "org.apache.iceberg.spark.SparkCatalog")
            .config(f"spark.sql.catalog.{cat}.type", "rest")
            .config(f"spark.sql.catalog.{cat}.uri", config.NESSIE_URI)
            .config(f"spark.sql.catalog.{cat}.warehouse", config.WAREHOUSE_URI)
            .config(f"spark.sql.catalog.{cat}.io-impl", "org.apache.iceberg.aws.s3.S3FileIO")
            .config(f"spark.sql.catalog.{cat}.s3.endpoint", config.S3_ENDPOINT)
            .config(f"spark.sql.catalog.{cat}.s3.path-style-access", "true")
            .config(f"spark.sql.catalog.{cat}.s3.access-key-id", config.S3_KEY)
            .config(f"spark.sql.catalog.{cat}.s3.secret-access-key", config.S3_SECRET)
            .config(f"spark.sql.catalog.{cat}.client.region", config.S3_REGION)
            .config("spark.sql.defaultCatalog", cat)
        )
        spark = builder.getOrCreate()
        spark.sparkContext.setLogLevel("WARN")
        return spark

7. catalog.py — namespaces & write helpers

Maps each medallion layer to an isolated namespace and centralizes the three write patterns.

def namespace(layer): return f"auth_{layer}"                 # bronze -> auth_bronze
def table_fqn(layer, name): return f"nessie.auth_{layer}.{name}"
def ensure_namespace(spark, layer):
    spark.sql(f"CREATE NAMESPACE IF NOT EXISTS nessie.auth_{layer}")

def table_exists(spark, fqn) -> bool:        # reliable across the REST catalog
    try:    spark.table(fqn); return True
    except Exception: return False

write_iceberg — bronze writes snapshot / append

def write_iceberg(df, fqn, partition_cols, mode):
    if not table_exists(spark, fqn):
        writer = df.writeTo(fqn).using("iceberg")
        if partition_cols: writer = writer.partitionedBy(col(partition_cols[0]), ...)
        writer.create()                 # first run: create from df
        return
    if   mode == "snapshot": df.writeTo(fqn).overwritePartitions()   # reload this day's partition
    elif mode == "append":   df.writeTo(fqn).append()                # add the day's new events
  • snapshot (reference/dimension-like tables: users, roles, …) — overwritePartitions() dynamically replaces only the logical_date partitions present in the dataframe, so re-running a day is idempotent.
  • append (event tables: sessions, login_attempts, …) — adds the day's slice.

upsert_iceberg — silver SCD1 via MERGE INTO

def upsert_iceberg(df, fqn, keys):
    if not table_exists(spark, fqn):
        df.writeTo(fqn).using("iceberg").create(); return        # initial load
    df.createOrReplaceTempView(view)
    on = " AND ".join(f"t.{k} = s.{k}" for k in keys)
    spark.sql(f"""
        MERGE INTO {fqn} t USING {view} s ON {on}
        WHEN MATCHED THEN UPDATE SET *
        WHEN NOT MATCHED THEN INSERT *""")

8. meta.py — meta-column injectors

Industry-standard provenance & change-detection columns, added with pure Spark expressions.

def add_bronze_meta(df, logical_date, run_id):
    source_cols = df.columns
    return (df
      .withColumn("raw_payload", F.to_json(F.struct(*[F.col(c) for c in source_cols])))
      .withColumn("ingest_ts", F.current_timestamp())
      .withColumn("logical_date", F.to_date(F.lit(logical_date)))
      .withColumn("source", F.lit(SOURCE_SYSTEM))
      .withColumn("_ingest_run_id", F.lit(run_id))
      .withColumn("_source_system", F.lit(SOURCE_SYSTEM)))

def add_silver_meta(df, business_cols, logical_date):
    return (df
      .withColumn("_record_hash",
          F.sha2(F.concat_ws("||", *[F.coalesce(F.col(c).cast("string"), F.lit("")) for c in business_cols]), 256))
      .withColumn("_loaded_at", F.current_timestamp())
      .withColumn("_source_logical_date", F.to_date(F.lit(logical_date))))

raw_payload captures the original row as JSON (audit/replay). _record_hash is a NULL-safe SHA-256 over the business columns — it powers silver de-dup and the gold SCD2 change test.

10. Class design

A classic template method: the base orchestrates extract → transform → load; subclasses fill in the steps. Gold overrides run() because it builds one named dim/fact per call.

classDiagram class SparkJob { +layer: str +spark, table, logical_date, run_id +run() void +extract()* DataFrame +transform(df)* DataFrame +load(df)* void } class AuthBronzeJob { layer = "bronze" } class AuthSilverJob { layer = "silver" } class AuthGoldJob { layer = "gold"\n+run() overridden } SparkJob <|-- AuthBronzeJob SparkJob <|-- AuthSilverJob SparkJob <|-- AuthGoldJob
jobs/base.py
class SparkJob(ABC):
    layer = ""
    def __init__(self, spark, table, logical_date, run_id): ...
    def run(self):                              # template method
        catalog.ensure_namespace(self.spark, self.layer)
        df  = self.extract()
        out = self.transform(df)
        self.load(out)
        self.log_done(out)
    @abstractmethod
    def extract(self) -> DataFrame: ...
    @abstractmethod
    def transform(self, df) -> DataFrame: ...
    @abstractmethod
    def load(self, df) -> None: ...

11. Bronze job

Raw landing: JDBC read → add bronze meta → write Iceberg (snapshot or append).

class AuthBronzeJob(SparkJob):
    layer = "bronze"
    def extract(self):
        return (self.spark.read.format("jdbc")
            .option("url", config.jdbc_url())
            .option("dbtable", self.spec.select_sql(self.logical_date))
            .option("user", config.PG_USER).option("password", config.PG_PASSWORD)
            .option("driver", "org.postgresql.Driver").load())
    def transform(self, df):
        return meta.add_bronze_meta(df, self.logical_date, self.run_id)
    def load(self, df):
        fqn = catalog.table_fqn("bronze", self.spec.bronze)        # nessie.auth_bronze.<t>_raw
        mode = "snapshot" if self.spec.mode == "snapshot" else "append"
        catalog.write_iceberg(df, fqn, partition_cols=["logical_date"], mode=mode)

12. Silver job

Read the day's bronze slice → keep latest row per natural key → clean → add silver meta → SCD1 upsert.

def extract(self):
    df = self.spark.table(bronze).where(F.col("logical_date") == F.to_date(F.lit(self.logical_date)))

13. Gold job + SCD2

AuthGoldJob overrides run() and dispatches on the table name:

handler = {"dim_date":..., "dim_role":..., "dim_auth_method":...,
           "dim_user":..., "fact_login_attempts":..., "fact_sessions":...}[self.table]
  • dim_date — generated calendar (sequence(2020-01-01 … logical_date)), upsert on date_key.
  • dim_role — from silver roles + permission counts; SCD1 upsert.
  • dim_auth_method — distinct methods from sessions/logins, mapped to a family; SCD1.
  • dim_user — SCD Type 2 (below).
  • fact_login_attempts / fact_sessions — point-in-time FK to dim_user.

dim_user — Slowly Changing Dimension Type 2

The source is the current silver users enriched with is_mfa_enabled and primary_role. A SHA-256 _record_hash over the tracked attributes decides whether a user changed. The build is a two-step close-out + insert (a single MERGE cannot both close the old version and insert the new one for the same key):

# build src with _record_hash over SCD2_TRACKED_ATTRS and _effective_from = logical_date
# Step 1 — close out current rows whose hash changed
MERGE INTO nessie.auth_gold.dim_user t
USING _dim_user_src s
  ON t.user_nk = s.user_nk AND t._is_current = true
WHEN MATCHED AND t._record_hash <> s._record_hash THEN
  UPDATE SET t._is_current = false, t._effective_to = s._effective_from

# Step 2 — insert brand-new users + new versions of just-closed users
current = dim_user.where(_is_current)                       # after step 1
new_versions = src LEFT JOIN current ON user_nk
               WHERE current is null OR hash differs
new_versions.writeTo(dim_user).append()

Each version row carries: user_sk = md5(user_nk | _effective_from) (surrogate key), _effective_from, _effective_to (sentinel 9999-12-31 while current), and _is_current.

Facts — point-in-time dimension join

A fact row joins to the version of the user that was current when the event happened:

src.join(dim_user,
   (src.user_id == dim.user_nk)
   & (dim._effective_from <= src.attempted_at)
   & (src.attempted_at < dim._effective_to), "left")
# user_sk = coalesce(matched user_sk, "-1")   # "-1" = unknown member

Facts also derive date_key, auth_method_sk = md5(auth_method), measures (is_success, session_duration_seconds, …), and are partitioned by event date, upserted on their PK for idempotency.

16. __main__.py — entry point

_JOBS = {"bronze": AuthBronzeJob, "silver": AuthSilverJob, "gold": AuthGoldJob}

def main():
    args = parse_args()                       # --job --table --date [--run-id]
    run_id = args.run_id or f"manual-{uuid.uuid4()}"
    spark = SparkSessionFactory.build(f"auth_{args.job}_{args.table}_{args.date}")
    try:
        _JOBS[args.job](spark, args.table, args.date, run_id).run()
    finally:
        spark.stop()

Run as a module: python -m etl_app_spark --job bronze --table users --date 2026-05-01
