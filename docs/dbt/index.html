Overview

The rideon dbt project transforms raw OLTP data from the Rideon ride-hailing platform into a Kimball star schema (silver) and pre-aggregated analytics marts (gold), all persisted as Apache Iceberg tables inside the Nessie catalog.

Architecture

The project follows the medallion architecture: raw OLTP data lands in Bronze, is deduplicated through ephemeral Staging CTEs, structured into a Kimball star in Silver, and aggregated into business-facing Gold marts.

Layer Descriptions

LayerNamespaceMaterializationPurpose
Bronze nessie.bronze External (etl-app) Raw OLTP tables with full-row CDC appends. Managed by jobs/rideon_bronze, not by dbt.
Staging — Ephemeral CTE Dedup-to-latest-per-PK transforms using ROW_NUMBER() OVER (PARTITION BY pk ORDER BY ingest_ts DESC). Inlined into Silver — no views or tables created.
Silver nessie.silver Iceberg table Kimball star schema: 6 dimensions + 4 fact tables. Conformed dims are enriched with attributes from related staging models.
Gold nessie.gold Iceberg table Pre-aggregated analytics marts by business domain (city performance, driver earnings, rider LTV, surge pricing).

Infrastructure

Docker Image

Built from services/dbt/Dockerfile using python:3.11-slim. Tagged ${COMPOSE_PROJECT_NAME}-dbt:latest. ENTRYPOINT is dbt — callers pass subcommands directly (e.g. build --select path:models/silver).

Profiles are baked in at build time, templated from env vars at runtime. The docker-socket-proxy denies VOLUMES, so bind-mounts are not used.

Airflow Integration

The rideon_etl DAG spawns ephemeral containers via DockerOperator referencing the pre-built image. Silver and gold layers are built in separate operator tasks for dependency isolation.

The service is declared with profiles: ["build-only"] so it is excluded from make up — containers are on-demand only.

Staging Models

All 10 staging models are ephemeral — they compile into CTEs inside the Silver models. The shared pattern: ROW_NUMBER() OVER (PARTITION BY <pk> ORDER BY ingest_ts DESC) = 1 to keep the latest version of each record from the append-only bronze tables.

dim_date dimension table

Date spine generated dynamically from the observed range of requested_at in rides. Uses Trino's sequence() + unnest() — no external packages required.

ColumnTestNotes
date_dayunique · not_nullPK
year
quarter1–4
month1–12
dayDay of month
day_of_week1=Mon … 7=Sun
day_nameFull weekday name (e.g. Monday)
iso_weekISO week number
is_weekendBoolean; true for day_of_week ∈ {6, 7}
View SQL 
with bounds as (
    select
        cast(min(requested_at) as date) as min_d,
        cast(max(requested_at) as date) as max_d
    from {{ ref('stg_rideon__rides') }}
),
spine as (
    select d as date_day
    from bounds
    cross join unnest(sequence(bounds.min_d, bounds.max_d, interval '1' day)) as t(d)
)
select
    date_day,
    year(date_day)                           as year,
    quarter(date_day)                        as quarter,
    month(date_day)                          as month,
    day(date_day)                            as day,
    day_of_week(date_day)                    as day_of_week,
    date_format(date_day, '%W')              as day_name,
    week(date_day)                           as iso_week,
    day_of_week(date_day) in (6, 7)          as is_weekend
from spine

Silver — Facts

Four fact tables in nessie.silver, each table materialized as Iceberg. All include FK references to the six Silver dims and denormalized boolean flags for common filter patterns.

Grain: one row per ride. Central fact table. Joins rides with their fare breakdown; computes derived measures (wait_min, boolean flags).

with rides as (select * from {{ ref('stg_rideon__rides') }}),
fares as (
    select ride_id, total_fare, surge_amount, discount, currency
    from {{ ref('stg_rideon__fares') }}

select
    r.ride_id,
    cast(r.requested_at as date)                          as ride_date,
    r.rider_id, r.driver_id, r.vehicle_id, r.city_id, r.category_id,
    r.status, r.cancelled_by,

Gold Marts

Four pre-aggregated analytics marts in nessie.gold, each table materialized as Iceberg. Built from Silver dims and facts — no direct staging references.

daily_city_performance mart table

Grain: city × day. Demand, fulfilment rate, cancellation rate, revenue, and unique participant KPIs per city per day.

ColumnNotes
dayPartition date
city_id, city_name, country
total_rides
completed_rides
cancelled_rides
cancellation_rate_pct100 × cancelled / total
gross_fareSum of total_fare for completed rides
avg_surgeAverage surge_multiplier
avg_distance_kmCompleted rides only
avg_duration_minCompleted rides only
unique_riders
unique_drivers
View SQL 
with rides  as (select * from {{ ref('fact_rides') }}),
cities as (select * from {{ ref('dim_city') }})
select
    r.ride_date                                                   as day,
    r.city_id, c.city_name, c.country,
    count(*)                                                      as total_rides,
    count_if(r.is_completed)                                      as completed_rides,
    count_if(r.is_cancelled)                                      as cancelled_rides,
    round(100.0 * count_if(r.is_cancelled) / count(*), 2)       as cancellation_rate_pct,
    coalesce(sum(r.total_fare), 0)                               as gross_fare,
    round(avg(r.surge_multiplier), 3)                            as avg_surge,
    round(avg(case when r.is_completed then r.distance_km end), 2) as avg_distance_km,
    round(avg(case when r.is_completed then r.duration_min end), 2) as avg_duration_min,
    count(distinct r.rider_id)                                   as unique_riders,
    count(distinct r.driver_id)                                  as unique_drivers
from rides r
left join cities c on r.city_id = c.city_id
group by r.ride_date, r.city_id, c.city_name, c.country

with rides as (select * from {{ ref('fact_rides') }}),
payments as (select rider_id, amount, status from {{ ref('fact_ride_payments') }}),
riders  as (select * from {{ ref('dim_rider') }}),
ride_agg as (
    select
        rider_id,
        count(*)                                                     as total_rides,
        count_if(is_completed)                                       as completed_rides,
        count_if(is_cancelled)                                       as cancelled_rides,
        min(ride_date)                                               as first_ride_date,
        max(ride_date)                                               as last_ride_date,
        round(avg(case when is_completed then distance_km end), 2)  as avg_distance_km
    from rides group by rider_id
),
pay_agg as (
    select
        rider_id,
        coalesce(sum(case when status = 'captured' then amount end), 0) as total_spend,
        round(avg(case when status = 'captured' then amount end), 2)   as avg_fare
    from payments group by rider_id
)
select
    r.rider_id, rd.full_name as rider_name,
    rd.city_name, rd.country,
    r.total_rides, r.completed_rides, r.cancelled_rides,
    round(100.0 * r.cancelled_rides / nullif(r.total_rides, 0), 2) as cancellation_rate_pct,
    p.total_spend, p.avg_fare,
    r.avg_distance_km, r.first_ride_date, r.last_ride_date
from ride_agg r
left join pay_agg p  on r.rider_id = p.rider_id
left join riders rd  on r.rider_id = rd.rider_id

View SQL 
with rides      as (select * from {{ ref('fact_rides') }}),
cities     as (select city_id, city_name, country from {{ ref('dim_city') }}),
categories as (select category_id, code, display_name from {{ ref('dim_vehicle_category') }})
select
    r.ride_date                                               as day,
    r.city_id, c.city_name,
    r.category_id, cat.code                                   as category_code,
    count(*)                                                  as total_rides,
    count_if(r.is_surge)                                      as surge_rides,
    round(100.0 * count_if(r.is_surge) / count(*), 2)       as surge_ride_pct,
    round(avg(r.surge_multiplier), 3)                         as avg_surge_multiplier,
    round(max(r.surge_multiplier), 2)                         as max_surge_multiplier,
    coalesce(sum(r.surge_amount), 0)                          as surge_revenue,
    coalesce(sum(r.total_fare), 0)                            as gross_fare
from rides r
left join cities c     on r.city_id = c.city_id
left join categories cat on r.category_id = cat.category_id
group by r.ride_date, r.city_id, c.city_name, r.category_id, cat.code

Macros

generate_schema_name

Overrides dbt's default schema name generation. dbt's default concatenates the target schema and the custom schema (e.g. silver_gold), which creates the wrong Nessie namespace. This macro maps +schema: gold directly to the namespace gold, and falls back to the target schema when no custom schema is set.

ArgumentDescription
custom_schema_nameValue from +schema: in dbt_project.yml; none for models without an explicit schema
nodeThe dbt model node (unused in this implementation)
InputOutput
custom_schema_name = nonetarget.schema (e.g. silver)
custom_schema_name = "gold"gold
custom_schema_name = "silver"silver
